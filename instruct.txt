You are a senior backend architect and Python expert.

Build a production-grade REST API for an AI FinOps Platform using the following stack:

- Framework: FastAPI
- ORM: SQLAlchemy (already defined models will be provided)
- Database: PostgreSQL
- Auth: JWT-based authentication (multi-tenant aware)
- Validation: Pydantic v2
- Background Jobs: Celery or FastAPI BackgroundTasks
- Caching (optional): Redis
- Migrations: Alembic

### 🎯 Core Requirements

1. Multi-Tenant Architecture:
   - Every request must be scoped by tenant_id
   - Extract tenant_id from JWT token
   - Enforce row-level isolation

2. API Modules to Implement:

### 👤 Auth & Users
- POST /auth/login
- POST /auth/register
- GET /users/me
- CRUD for users (role-based: owner/admin/viewer)

### 🏢 Tenants
- CRUD tenants (only owner)
- Tenant usage summary API

### ☁️ Cloud Accounts
- CRUD cloud accounts
- Sync billing data trigger endpoint

### ⚙️ Kubernetes
- GET /clusters
- GET /nodes
- GET /pods
- Filters: status, region, cluster_id

### ⚡ Accelerators (GPU/TPU/LPU)
- CRUD accelerators
- GET utilization summary
- GET idle vs active cost

### 🤖 Agents
- CRUD agents
- Versioning support
- Activate / deactivate agent

### 🧠 Orchestration Jobs
- Create job
- Get job status
- List jobs with filters (status, time range, agent_id)
- DAG support (parent/child jobs)

### 📊 Metrics
- GET /metrics/utilization (hourly aggregation)
- GET /metrics/cost-trend
- Support time range queries

### 💰 Cost & Billing
- GET /cost/overview
- GET /cost/by-service
- GET /cost/by-job
- GET /cost/anomalies (basic anomaly detection logic)

### 📦 Job Cost Aggregation
- Trigger aggregation
- Fetch job cost breakdown

---

### ⚙️ Technical Expectations

- Use clean architecture:
  - routers/
  - services/
  - repositories/
  - schemas/
  - core/

- Implement:
  - Dependency Injection
  - DB session management
  - Exception handling middleware
  - Logging (structured logs)
  - Pagination & filtering
  - Rate limiting (optional)

- Security:
  - JWT auth (access + refresh tokens)
  - Role-based access control (RBAC)
  - Tenant-level data isolation

- Performance:
  - Use indexes efficiently
  - Avoid N+1 queries
  - Use async where possible

---

### 📊 Bonus Features (Important)

- Cost anomaly detection (simple z-score or threshold-based)
- Idle resource detection (based on utilization < threshold)
- Cost per token calculation for LLM jobs
- Top expensive jobs API
- GPU utilization heatmap endpoint

1. Project folder structure

4. DB connection setup
5. Auth implementation
6. At least 2 fully implemented modules (Jobs + Cost)
7. Sample API requests (curl or Postman)
8. Comments explaining key design decisions

Use the provided SQLAlchemy ORM models exactly as given (do not modify schema unless necessary).

- Follow best practices used in scalable SaaS platforms
- Keep it modular and extensible
